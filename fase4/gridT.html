CSS Grid

Recordemos que CSS es conocido como Hoja de Estilo en Cascada (Cascading Sheet Style) y estas aparecieron queriendo separar el estilo de la página con el contenido de esta, es decir que se depende mucho de como el navegador va interpretando la forma en que se va a visualizar la página. Las hojas de estilo se pueden clasificar de dos formas:

Según como se apliquen en el código

  • Hoja Interna: Solo afecta al contenido de la página donde se crea
  • Hoja Externa: Afecta a todos los documentos que se víncules a ella

Según como afectan el contenido
  • Clase:Es el estilo que aplica a toda la clase donde se este referenciando
  • Etiqueta: Cuando se define cual va a ser el contenido sobre una etiqueta
  • Identificador Id: Se marca en la etiqueta cual es el estilo

Dentro de las etiquetas más importantes que tenemos es la STYLE la cual es declarada en la cabecera de la cabecera del documento. Por otro lado, recordemos que lo que se quiere con esto es crear una especie de rejilla, por tal motivo vamos a disponer de espacios tanto verticales y horizontales. Por tal motivo es que aqui vamos a tener dos contenedores, la rejilla y elemento, estos elementos se pueden organizar de una forma fácil a traves de nuestra etiqueta DIV la cual le podremos decir si el contenido que vamos a añadir va a ingresar una nueva fila o columna y si lo que buscamos es tener una mejor estructura necesitamos utlizar las etiquetas de estructuración semántica más importantes las cuales son de Header, Nav y Footer, Setion, Article y Aside. Una de sus ventajas es que estas rejillas se pueden ajustar automáticamente y la frecuencia con la que se debe repetir el esquema para esto debemos tener en cuenta un concepto que vamos a llamar media Query.
